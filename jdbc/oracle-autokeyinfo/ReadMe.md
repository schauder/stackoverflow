## Reproducer for the "autoKeyInfo is not initialized" problem.

### Description

When trying to perform batch inserts in a table with generated id the oracle JDBC driver throws `java.lang.AssertionError: autoKeyInfo is not initialized`.

The same does work with an older version of the ojdbc8 driver (19.15.0.0.1), but fails with the latest (21.6.0.0.1) versions of both ojdbc8 and ojdbc11.

ojdbc8 21.1.0.0 throws a different exception: `operation not allowed: DML Returning cannot be batched`


### Environment

I ran the tests on MacOS 12.3
and with Zulu JDK 17.0.0

Database is a docker container created with Testcontainers: gvenzl/oracle-xe:21.3.0-slim 

### Run the reproducer

For reproducing the issue execute the following script:

```
./run-all.sh
```
It will execute the following three builds.
```
./mvnw clean verify -Dojdbc8 -Dojdbc.version=19.15.0.0.1
./mvnw clean verify -Dojdbc8 -Dojdbc.version=21.1.0.0
./mvnw clean verify -Dojdbc8 -Dojdbc.version=21.3.0.0
./mvnw clean verify -Dojdbc8
./mvnw clean verify
```
The first one uses ojdbc8 version 19.15.0.0.1 and succeeds.

The second one uses ojdbc8 version 21.1.0.0 and fails with '' error. See second stacktrace below.

The final three use ojdbc8 or ojdbc11 with version 21.3.0.0 or 21.6.0.0.1 and fail.
